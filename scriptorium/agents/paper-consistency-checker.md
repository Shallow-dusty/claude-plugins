# 论文数据一致性审核 Agent

## 目标

交叉验证论文 (`thesis/`) 中所有数值声明与实验结果 (`results/`) 的一致性，输出不一致报告。

## 审核范围

### 数据源 (Source of Truth)

1. **实验结果 JSON**: `results/supplementary/test_set_results.json` — 测试集评估的权威数据
2. **训练日志**: `results/` 和 `results_3070_run1/`、`results_rerun_3070/` 下的 CSV/日志
3. **模型信息**: `results/supplementary/summary/` 下的汇总文件

### 验证目标 (Claims to Verify)

1. **thesis/data/chapter-5.tex** — 实验结果章节（核心表格和数值）
2. **thesis/data/chapter-6.tex** — 结论章节（总结性数值声明）
3. **thesis/data/abstract-cn.tex** 和 **abstract-en.tex** — 摘要中的数值
4. **thesis/data/appendix.tex** — 附录中的重跑验证数据

### 关键数值清单

对每个实验 (E0-E7) 检查：
- mAP@0.5（验证集 / 测试集）
- mAP@0.5:0.95（验证集 / 测试集）
- Precision / Recall / F1
- 推理速度 (ms/FPS)
- 参数量 / 参数增量
- 各改进的百分点提升量（如 "+1.2pp"）

## 执行步骤

### Step 1: 加载实验数据

读取 `results/supplementary/test_set_results.json`，解析每个实验的测试集指标。

### Step 2: 扫描论文数值

用 Grep 扫描 `thesis/data/*.tex` 中所有包含数值的行：
- 百分比模式: `\d+\.\d+%` 或 `\d+\.\d+\\%`
- mAP 相关: `mAP.*0\.\d+` 或数值如 `71.8`、`73.0`、`46.5`
- 参数量: `\d+\s*个参数` 或 `\d+\.\d+M`
- 速度: `\d+\.\d+\s*ms` 或 `\d+\s*FPS`
- 百分点提升: `[+\-]\d+\.\d+.*百分点` 或 `percentage point`

### Step 3: 交叉验证

对每个提取到的数值声明：
1. 确定其所属实验 (E0-E7)
2. 确定其指标类型 (mAP50/P/R/速度等)
3. 与 Step 1 的数据比对
4. 计算提升量是否正确（如 E2 mAP50 - E0 mAP50 = 声明的提升量）

### Step 4: 检查内部一致性

- 摘要、正文、结论中对同一指标的描述是否一致
- 表格数据与文字描述是否一致
- 中英文摘要数值是否一致

## 输出格式

```markdown
# 论文数据一致性审核报告

## 审核时间: [日期]

## 一致性验证结果

### 通过项 (数值一致)
| 位置 | 声明 | 实际值 | 状态 |
|------|------|--------|------|

### 不一致项 (需要修正)
| 位置 | 声明值 | 实际值 | 差异 | 严重程度 |
|------|--------|--------|------|----------|

### 内部一致性
| 指标 | 摘要(中) | 摘要(英) | 正文 | 结论 | 一致? |
|------|----------|----------|------|------|-------|

## 建议修正
- [具体修改建议]
```

## 注意事项

- 论文中报告的验证集指标 (val) 和测试集指标 (test) 要区分清楚
- 百分比和小数转换要注意（0.730 = 73.0%）
- 四舍五入可能导致小数点后最后一位不同，差异 < 0.1% 可标记为 "舍入差异" 而非 "不一致"
- 论文中有两套数据：原始训练 (Kaggle/3070 混合) 和 3070 独立重跑，注意区分
